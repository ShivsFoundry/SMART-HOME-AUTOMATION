# Security Policy

## Supported Versions

This project is actively maintained for educational and demonstration purposes. Security updates are generally applied to the latest version of the project.

| Version | Supported |
|---------|-----------|
| Latest | ✅ Yes |
| Older Versions | ❌ No |

---

## Reporting a Vulnerability

If you discover a security vulnerability or any issue that could affect the safe operation of this project, please report it responsibly.

When reporting an issue, please include:

- A clear description of the vulnerability.
- Steps to reproduce the issue.
- The affected Arduino board or hardware.
- Arduino IDE version.
- Any relevant screenshots, logs, or error messages.
- Suggested fixes (if available).

Please **do not publicly disclose** security vulnerabilities until they have been reviewed.

---

## Scope

This repository contains educational Arduino code and Tinkercad simulation files. Security reports are generally applicable to:

- Arduino source code
- Circuit design
- Hardware wiring documentation
- Project documentation

The following are **not considered security vulnerabilities**:

- Syntax errors
- Compilation errors
- Wiring mistakes made outside this repository
- Hardware failures
- Issues caused by unsupported Arduino libraries
- Tinkercad simulator limitations

---

## Responsible Disclosure

Please report vulnerabilities privately through GitHub by using:

**Security → Report a vulnerability**

or by opening a **private security advisory**, if available.

If private reporting is unavailable, you may contact the repository owner through GitHub before creating a public issue.

---

## Response Process

After receiving a report, the following process will be followed:

1. Acknowledge the report.
2. Review and reproduce the issue.
3. Evaluate the impact.
4. Develop and test a fix.
5. Release an updated version if necessary.
6. Credit the reporter (if they wish to be acknowledged).

---

## Best Practices

When using this project:

- Verify all wiring before powering the circuit.
- Use the correct operating voltage for all components.
- Test the project in Tinkercad before assembling physical hardware.
- Never connect high-voltage AC devices directly to the Arduino.
- Use a suitable transistor or relay driver when controlling motors or relays.
- Follow proper electrical safety practices during hardware implementation.

---

## Disclaimer

This project is intended for **educational and learning purposes**. While reasonable efforts have been made to provide accurate code and documentation, users are responsible for verifying the safety and correctness of their own hardware implementations.
